## Key Learnings

- **Postgres should be a derived index for content-heavy layers** —
  spec §14 said it explicitly; implementation drifted to `body_md
  TEXT`. At scale TEXT in PG = TOAST overflow + VACUUM cost + huge
  backups + replication lag. Always split tier-1 (hot scalar/indexed)
  from tier-4 (cold blob). Django's `FileField` makes this trivial
  via `default_storage`.

- **"Embed full body" is the wrong default** — BGE-small max context
  is 512 tokens (~1900 chars). For anything longer, naïve truncation
  loses critical signal (contract signatures at the end, meeting
  decisions late, ADR consequences last). Solution: per-type
  **sampling strategy** (`fixed_window` / `head_heavy` / `head_tail`
  / `uniform`) declared on TypeSpec. ~$0 cost; same call shape.

- **Three different questions need three different reverse backlinks**
  — `applied_in` (append list, "who cites me"), `superseded_by`
  (scalar UUID, "what replaced me"), `contradicts` (symmetric list,
  "what disagrees"). Forcing one shape would compromise all three.
  The other 6 forward edges (entity_refs / cross_refs / parent /
  related) are derived at read via GIN containment — cheap and
  unbounded.

- **PageIndex maps onto our `parent` edge already** — VectifyAI's
  reasoning-RAG approach (LLM walks tree of section summaries) is
  conceptually a tree of `CortexEntity` with `parent` set. We don't
  need their library; we need their pattern. Example: section
  entities under a doc entity, each with HaikuFitter-generated
  `summary`. Agent reads titles + summaries, recurses to leaves.

- **Per-type chunking thresholds beat one-size-fits-all** — chats,
  emails, meetings, notes fit a single 384-dim vector (single sampler
  call). Long docs need two-tier (doc + chunks) or three-tier (doc +
  sections + chunks). Threshold-gated via TypeSpec
  (`long_threshold_bytes`, `section_tier_threshold_bytes`,
  `chunk_strategy`).

- **`get_or_create` doesn't play with FileField cleanly** — FileField
  needs the row's PK first (so `upload_to` callable can use
  `instance.id`). Refactored `DeterministicResolver._spawn` to
  fast-path existing rows via `filter().first()`, then `save()` +
  `body.save(ContentFile(...))` for new ones.

## Solutions & Fixes

- **`docupal.core.*` imports across OCR + LLM modules** → blanket
  rename to `donna.core.*` via `replace_all` Edit. Root cause: code
  copied from sibling project (Docupal) without ns fix. Affected:
  `donna/core/ocr/{__init__,easyocr_,markitdown_,pymupdf4llm_,
  docling_,llm}.py`, `donna/core/llm/{factory,provider,tools,
  interface,utils,example_tools,handlers/base}.py`. Plus stubbed
  missing `donna.core.observability.decorators.observe_llm` (no-op
  passthrough) + `donna.core.ocr.image_enhancement.enhance_for_ocr`
  (raise NotImplementedError).

- **`observe_llm()` called with no args = decorator factory** →
  rewrote stub to support both `@observe_llm` and `@observe_llm()`
  call shapes. Pattern: introspect `dargs[0]` — if callable + no
  kwargs, bare-decorator; else parameterised factory.

- **Postgres image had no pgvector binaries** → swapped
  `postgres:17-alpine` → `pgvector/pgvector:pg17` in
  `docker-compose.yml`. Same PG17 wire protocol + on-disk format →
  existing volume mounted unchanged.

- **`body_md` as PG TEXT column** → replaced with
  `models.FileField(upload_to=_body_upload_path,
  storage=default_storage)` + `body_byte_size` IntegerField + new
  `load_body()` lazy fetch. Path: `cortex/<workspace_id>/<type>/
  <entity_id>.md`. Migration 0001 rewritten in place (no prod data).
  Pipeline step 11 saves entity first (no body), then
  `entity.body.save(ContentFile(body_bytes), save=True)`, then
  reverse-edge updates — all inside one `transaction.atomic()`.

- **Sampler budget arithmetic off-by-N** → `fixed_window_sampler`
  emitted 1902 chars vs 1900 cap because budget didn't account for
  16 chars of separators (`\n[...]\n` × 2). Re-derived budget:
  `max_chars - len(head) - 2 * len(sep)`. Same fix in
  `head_heavy_sampler`, `head_tail_sampler`, `uniform_sampler`.

- **Linter couldn't read body pre-persist** — body lives in FileField
  which is set only after save. Fix: `_check_source_footer(entity,
  body_md=None)` accepts an in-memory body arg; pipeline step 10
  passes `body_md_final` explicitly. Post-persist callers fall
  back to `entity.load_body()`.

- **Reaped-orphan reverse path was unreliable** — for nightly task,
  walk `default_storage.listdir(...)` recursively; for each
  `<uuid>.md` check `CortexEntity.objects.filter(id=uuid).exists()`;
  delete if missing. Added to beat schedule.

## Decisions Made

- **Storage abstraction = Django `FileField` + `default_storage`** —
  chose over raw `VARCHAR(500) body_storage_key` because FileField
  ships `.url` (presigned S3), `.size`, `.delete()`, `.open()`
  streaming, and storage backend abstraction free. Existing
  `DONNA_STORAGE_BACKEND=s3|filesystem|gcs|azure` env toggle keeps
  working. Trade: slightly more Django magic vs explicit.

- **Per-type embedding sampler** (P0.14) — `meeting=uniform`,
  `email/chat/ticket=head_heavy`, `doc=head_tail`, rest default
  `fixed_window`. Wired via new `TypeSpec.embedding_sampler` field
  (default `fixed_window_sampler`). Rationale documented in plan
  doc `06 - status/04-p0.14-...md`.

- **`--rebuild` flag in `cortex_sync` blocked in v1** — current
  Jinja templates emit display-grade frontmatter only (title,
  occurred_at, parent_path, slug, type extensions). Don't carry
  edges or provenance. A files-as-truth rebuild today would silently
  drop edge data. Command raises CommandError pointing at P10 (Mode
  A vault projection) as the unblocker. Honest > silently broken.

- **Drive connector skipped** in P8 per user scope — only Gmail wired
  this round. Drive deferred until first client demand.

- **PageIndex** (VectifyAI long-doc RAG) NOT imported as dep —
  pattern (tree of summarised section nodes) maps onto our existing
  `parent` edge field. Implement natively when P0.15 activates.
  Library would add Python deps + their own parsing assumptions.

- **Scope extended past 40-page cap** — user explicitly chose to ship
  long-doc support now rather than defer. Activated P0.15 plan:
  three-tier (doc + section + chunk) with tree-walk mode for
  multi-hop reasoning. New plan doc at
  `06 - status/06-p0.15-long-document-support.md`; deferred
  predecessor marked SUPERSEDED.

- **Schema for long docs:** new `cortex_sections` table (chapter
  embeddings + LLM summaries) + `cortex_chunks` table (chunk
  embeddings + byte-range pointers into the FileField body).
  Three-tier ANN: doc → section → chunk. Tree-walk mode opt-in
  per query.

- **Section summaries deferred at write time** — section embeddings
  use title + first 500 chars instead, no Haiku calls per ingest.
  Summarisation runs lazily in a background pass post-v1. Keeps
  P0.15a write path zero-LLM.

- **Body in storage, edges in PG** — final tier split: hot scalar +
  indexed JSONB edge arrays stay in PG; cold large text moves to
  SilverStorage. Embeddings stay in PG (1.5KB each — hot).

## Errors & Workarounds

- **`ModuleNotFoundError: No module named 'litellm'`** when loading
  `donna.core.llm.factory`. Cause: dep not in pyproject. Fix: added
  `litellm>=1.55.0` + `python-dotenv>=1.0.0` to `pyproject.toml`,
  ran `uv sync`.

- **`ModuleNotFoundError: No module named 'pgvector'`** during
  `manage.py migrate cortex`. Cause: dep missing. Fix: added
  `pgvector>=0.3.0` to pyproject + `uv sync`.

- **`server closed the connection unexpectedly`** when host venv
  connecting to `localhost:5551`. Cause: `.env` set
  `DATABASE_PORT=5551` for both sides, but inside container PG
  actually listens on 5432. Worked around by running tests INSIDE
  the docker container (`docker exec donna-server python manage.py
  test ...`) — uses internal `database:5432` connection.

- **`pg_available_extensions` empty for vector** with
  `postgres:17-alpine`. Cause: alpine PG image lacks pgvector
  binaries. Fix: swapped to `pgvector/pgvector:pg17`. Existing PG17
  volume mounted unchanged.

- **`TypeError: observe_llm() missing 1 required positional argument:
  'fn'`** at `donna/core/llm/provider.py:619`. Cause: stub only
  supported bare `@observe_llm`, not `@observe_llm()`. Fix: rewrote
  stub to handle both shapes.

- **`AssertionError: 1902 not less than or equal to 1900`** in
  `test_fixed_window_long_body_within_budget`. Cause: separator-byte
  budget off-by-4. Fix: derived budget = `max_chars - head_len - 2
  * len(sep)` so total assembled string fits exactly.

- **`AssertionError: False is not true` on `out.endswith("alpha")`**
  after first sampler budget fix used `[:max_chars]` slice. Cause:
  truncation cut mid-word. Fix: dropped slice; correct budget
  arithmetic avoids over-build.

- **`LinterError: MISSING_SYNTHESIZED_AT`** during pipeline test
  after P0.5-P0.13 refactor. Cause: linter R2 required `created_at`
  to be set pre-save, but `TimestampsMixin.auto_now_add` only fires
  ON save. Fix: drop the pre-save check; `synthesized_at` is auto-
  populated; linter only verifies `occurred_at`.

- **`bfs: error: plans/cortex: No such file or directory`** when
  running `find plans/cortex` from cwd `donna/`. Worked around by
  using absolute path `/Users/ristoc/Workspaces/cube/donna/server/
  plans/cortex`.
