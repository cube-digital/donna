# P0.14 — Body to FileField + Sampled Embeddings

**Status:** planned · **Effort:** ~1 day · **Lands before:** P8, P9, P10

Two coupled refactors that fix the largest architectural drift in the
current implementation:

1. **Storage:** move `body_md` out of `cortex_entities` (Postgres TEXT
   column) into `SilverStorage` (S3 / filesystem / GCS / Azure) via
   Django `FileField`. Postgres becomes a derived index per spec §14.
2. **Embedding:** stop sending the full body to the embedder. Build a
   sampled representation (title + intro + middle + tail) that fits
   BGE-small's 512-token context window cleanly.

Document chunking (multi-tier embeddings for long docs) is **deferred**
to P0.15 — see [`05-deferred-document-chunking.md`](./05-deferred-document-chunking.md).

## Why now

| Drift | Cost at scale |
|---|---|
| `body_md` TEXT in Postgres | TOAST overflow + slow VACUUM + huge backups + replication lag |
| Full-body embedding | wastes BGE-small context window; truncation arbitrary |
| Each row's content tied to PG | `DROP TABLE` loses everything; spec §14 said opposite |

Each row currently stores full markdown in a PG `TEXT` column. Per
spec §14 (`"Postgres = derived index, dispensable. Files = truth."`)
the body belongs in `SilverStorage` and Postgres holds only metadata
+ pointer. That mismatch costs nothing today (test fixtures) and
catastrophic at production scale (millions of rows × KB-MB each).

## Storage refactor

### Model change

```python
# donna/cortex/models.py
from django.core.files.storage import default_storage


def _body_upload_path(instance: "CortexEntity", filename: str) -> str:
    """Canonical body path under SilverStorage.

    Mirrors spec §9 layout; one file per entity.
    """
    return f"cortex/{instance.workspace_id}/{instance.type}/{instance.id}.md"


class CortexEntity(TimestampsMixin):
    ...

    # REPLACES the old `body_md = TextField(...)`
    body = models.FileField(
        _("body markdown"),
        upload_to=_body_upload_path,
        storage=default_storage,
        max_length=500,
        help_text=_(
            "Markdown body stored in SilverStorage (S3 / filesystem / etc — "
            "per STORAGES['default'] config). Lazy-loaded via `load_body()`."
        ),
    )

    body_byte_size = models.IntegerField(
        _("body byte size"),
        default=0,
        help_text=_("Cheap byte-size stat — avoids opening the file just to count."),
    )

    def load_body(self) -> str:
        """Lazy fetch of the markdown body."""
        with self.body.open("rb") as f:
            return f.read().decode("utf-8")
```

### Why `FileField` over raw `VARCHAR(500)`

| Approach | Storage abstraction | URL gen | Stream | Delete | Code volume |
|---|---|---|---|---|---|
| `body_storage_key VARCHAR(500)` + manual `default_storage.open()` | manual | manual | manual | manual | more |
| `models.FileField(storage=default_storage, upload_to=...)` | automatic | `.url` free | `.open()` free | `.delete()` free | less |

`FileField` was designed for exactly this. It honours
`STORAGES["default"]` so the existing `DONNA_STORAGE_BACKEND` env-var
toggle (filesystem / s3 / gcs / azure) keeps working unchanged.

### Migration shape

```python

    # 4. Drop old TEXT column
    migrations.RemoveField(model_name="cortexentity", name="body_md"),

```

Data migration callable:

```python
def _migrate_bodies_to_storage(apps, schema_editor):
    CortexEntity = apps.get_model("cortex", "CortexEntity")
    from django.core.files.base import ContentFile
    for entity in CortexEntity.objects.all().iterator(chunk_size=500):
        body_bytes = entity.body_md.encode("utf-8")
        entity.body.save(
            name=f"{entity.id}.md",
            content=ContentFile(body_bytes),
            save=False,
        )
        entity.body_byte_size = len(body_bytes)
        entity.save(update_fields=["body", "body_byte_size"])
```

Test fixtures: empty workspace → migration is a no-op. For real data
it's chunked + idempotent.

### Pipeline step 11 — atomic persist

```python
# donna/cortex/pipeline.py — step 11 update
from django.core.files.base import ContentFile

with transaction.atomic():
    # 1. Write the body to SilverStorage first (under the entity's UUID path)
    new_entity.save()                   # INSERT row (without body — FileField wants id)
    new_entity.body.save(
        name=f"{new_entity.id}.md",
        content=ContentFile(body_md_final.encode("utf-8")),
        save=True,                       # writes FileField pointer + bumps updated_at
    )

    # 2. Reverse-edge updates (same atomic block)
    for target_id in new_entity.sources:
        self._append_applied_in(target_id, new_entity.id)
    for target_id in new_entity.supersedes:
        self._assign_superseded_by(target_id, new_entity.id)
    for target_id in new_entity.contradicts:
        self._append_contradicts(target_id, new_entity.id)
```

Failure ordering:

| Failure point | State after rollback |
|---|---|
| Storage write fails | row exists with empty `body` field — rolled back by atomic block |
| Reverse-edge write fails | entity + body rolled back; storage file orphaned (reaped nightly) |

### What stays in Postgres

| Column tier | Why |
|---|---|
| Identity (id, type, author, source, content_hash, occurred_at) | hot, indexed, small |
| Scope (workspace_id, client_id, project_id) | hot, indexed |
| Topical (cluster_id, doc_embedding) | hot, vector index |
| Edges (entity_refs, sources, applied_in, …) | hot JSONB arrays of UUIDs (bounded) |
| Extensions JSONB | hot per-type frontmatter (usually <1KB) |
| Title | hot, short |
| body_byte_size | hot stat |
| body (FileField pointer) | hot — but the file content is COLD in SilverStorage |

Everything queried frequently stays in PG. The body content — large,
read rarely (only during synthesis or rendering) — lives in S3.

## Embedding refactor

### New helper — sampled representation

```python
# donna/cortex/embeddings.py

def fixed_window_sampler(
    title: str,
    body_md: str,
    *,
    max_chars: int = 1900,
) -> str:
    """Default sampler. Window mix: title + intro + middle + tail.

    BGE-small max context = 512 tokens ≈ ~1900 chars EN. Spend the
    budget on the most informative regions:

    - title             (~100 chars)  identity anchor
    - intro             (~700 chars)  what this is
    - middle (centred)  (~600 chars)  what's discussed
    - tail              (~500 chars)  decisions / conclusion / signatures
    """
    if len(body_md) <= max_chars - len(title) - 4:
        return f"{title}\n\n{body_md}"

    intro = body_md[:700]
    mid_start = max(0, len(body_md) // 2 - 300)
    middle = body_md[mid_start : mid_start + 600]
    tail = body_md[-500:]
    return f"{title}\n\n{intro}\n[...]\n{middle}\n[...]\n{tail}"
```

### Per-type sampling override

Add `embedding_sampler` field to TypeSpec; default = `fixed_window_sampler`.

```python

```

Ship a small library of samplers:

```python
# donna/cortex/embeddings.py — additional samplers

def head_heavy_sampler(title, body_md, head_pct=0.7, mid_pct=0.2, tail_pct=0.1, max_chars=1900):
    """Weight head heavily — for chats (latest message at top) + emails."""

def head_tail_sampler(title, body_md, head_pct=0.3, tail_pct=0.7, max_chars=1900):
    """Weight tail heavily — for contracts (signatures + addendums at end)."""

def uniform_sampler(title, body_md, max_chars=1900):
    """Even sampling across N windows — for runbooks and meetings."""
```

```python
    type="email",
    ...
    embedding_sampler=head_heavy_sampler,

```

Cheat-sheet:

| Type | Sampler | Why |
|---|---|---|
| chat | `head_heavy_sampler` | latest message often at top of rendered output |
| email | `head_heavy_sampler` | latest reply at top |
| meeting | `uniform_sampler` | decisions distributed; sometimes at the end |
| doc | `head_tail_sampler` | signatures + addendums at end |
| ticket | `head_heavy_sampler` | issue summary first |
| clip | `fixed_window_sampler` (default) | unknown shape |
| note | `fixed_window_sampler` | usually short anyway |
| person/org/project/concept/decision | `fixed_window_sampler` | structured short rows |

### Embedder wrapper

```python
# donna/cortex/embeddings.py — wrapper

class BGESmallEmbedder:
    ...
    def embed_entity(
        self, title: str, body_md: str,
        sampler: Callable[[str, str], str] | None = None,
    ) -> list[float]:
        """Embed using a per-type sampled representation."""
        sampler = sampler or fixed_window_sampler
        text = sampler(title, body_md)
        return self.embed(text)
```

## Acceptance criteria

- `cortex_entities` Postgres table no longer has `body_md TEXT` column
- `entity.body.url` returns presigned S3 URL (when S3 backend) or
  filesystem URL (when filesystem backend) without code changes —
  matrix of backends already supported by `default_storage`
- `entity.body_byte_size` populated on write
- `entity.load_body()` returns the full markdown lazily
- All 11 existing tests still pass
- 1 new test: assert `body_md_final` content is in S3 file, NOT in
  `body_md` PG column (which doesn't exist anymore)
- 1 new test: assert sampled embedding input ≤ 1900 chars for a
  50KB body
- 1 new test: assert `head_heavy_sampler` produces a different
  embedding input than `head_tail_sampler` on the same body
- Orphan reaper: smoke test with manually-created orphan file
